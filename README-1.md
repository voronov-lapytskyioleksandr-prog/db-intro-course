
### Лабораторна робота 1: Збір вимог та розробка схеми ER

1. **Короткий виклад вимог**
Потреби зацікавлених сторін: Адміністратори повинні мати можливість додавати категорії страв, самі страви та їх склад. Офіціанти повинні мати можливість фіксувати замовлення клієнтів на певні столики.
Дані для зберігання: Інформація про категорії меню, страви (назва, ціна, вага), інгредієнти, а також дані про замовлення (дата, номер столика, загальна сума) та деталі замовлень (які саме страви замовлені).
Бізнес-правила:

* Кожна страва повинна належати до однієї категорії.
* Страва може складатися з багатьох інгредієнтів, а один інгредієнт може входити до багатьох страв.
* Одне замовлення може містити багато страв, і одна страва може бути присутня в багатьох різних замовленнях.

**2. Список сутностей з атрибутами та зв'язки**

Сутності та їх атрибути:

Category (Категорія):

* CategoryID (Primary Key)
* Name
* Description

Dish (Страва):

* DishID (Primary Key)
* Name
* Description
* Price
* Weight
* CategoryID (Foreign Key)

Ingredient (Інгредієнт):

* IngredientID (Primary Key)
* Name
* IsAllergen (чи є алергеном)

DishIngredient (Склад страви - асоціативна сутність):

* DishID (Foreign Key)
* IngredientID (Foreign Key)
* Primary Key: DishID, IngredientID

RestaurantOrder (Замовлення):

* OrderID (Primary Key)
* OrderDate
* TableNumber
* TotalAmount

OrderItem (Елемент замовлення - асоціативна сутність):

* OrderID (Foreign Key)
* DishID (Foreign Key)
* Quantity
* Primary Key: OrderID, DishID

Пояснення зв'язків:
Category та Dish: Зв'язок "один до багатьох". Одна категорія містить багато страв, але кожна страва належить лише до однієї категорії.
Dish та Ingredient: Зв'язок "багато до багатьох". Одна страва має багато інгредієнтів, і кожен інгредієнт використовується в багатьох стравах. Зв'язок реалізується через сутність DishIngredient.
RestaurantOrder та Dish: Зв'язок "багато до багатьох". Одне замовлення містить багато страв, і одна страва може бути замовлена багато разів. Зв'язок реалізується через сутність OrderItem.

**3. Припущення та обмеження**

* Ціна страви (Price) не може бути від'ємною.
* Кількість замовлених страв (Quantity) повинна бути більше нуля.
* Кожне замовлення обов'язково прив'язується до конкретного столика (TableNumber).
